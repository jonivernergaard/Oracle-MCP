Table: B610F/IIM (inv100) - Items
IID: 'IM' – active item, 'IZ' – inactive
IPROD: item
IDESC: item description
IVULP: quantity per pallet
IMCQTY: quantity per carton
IMNNWU: weight
IREF01-05: five reference fields (second screen inv100)
IMTCCD: parent for item
ICLAS: item class – USED FOR ACCOUNTING!!!
IITYP: item type
IUMR: unit of measure
IMABRV: CMS abbreviation – USED FOR ACCOUNTING!!!
IMCNTT: packing (arrangement) on pallet
IFII: quantity in bag
IFCI: number of bags

B610F/ETYK - Label Data (KWHS and then UPDETYK)
ITEMBPCS: item from BPCS
ITEMCUST: customer item
EANITEM: EAN for item (carton or single piece)
EANINNER: EAN for inner pack
LANG1-5: descriptions in five languages
PACKING: item packing e.g., 50 x 30
LOGO: optional logo on label
DESC1: description1 30 characters
DESC2: description2 30 characters
Relations:
IIM: ITEMBPCS=IPROD

Table: B610F/RCM (acr100) - Customers
CMID: active or inactive customer
CCUST: customer number e.g., 123
CNME: customer description e.g., "Makro Gdynia store no. 34"
CTYPE: customer type - USED FOR ACCOUNTING!!!
CMALPH: abbreviated name – alphanumeric e.g., "MAKRO"
CCCUS: parent customer number e.g., 100098 which is Makro Warsaw headquarters number
CDEA1: Key account
CDEA2: stores information whether customer is PROCESSING
CREF01-5: five reference fields
CMFF01-4: accounting fields – USED FOR ACCOUNTING!!!
CCOUN: country, but IC companies have e.g., PDA (meaning Germany DE)
CCURR: currency
CMBANK: bank

AVM (acp100) - Vendors
VMID: active or inactive vendor
VENDOR: vendor number
VNDNAM: vendor name
VTYPE: vendor type - USED FOR ACCOUNTING!!!
VNALPH: abbreviated name – alphanumeric
VCOUN: country
VMBANK: bank
VCURR: currency
VMREF1-5: reference fields, WMREF2 USED FOR ACCOUNTING!!!!
